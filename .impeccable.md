## Design Context

### Users

Primary users are SMB and solo marketers focused on e-commerce ad production, especially for fast-turnaround video and image creatives.
They typically work under time and budget constraints, need campaign-ready assets quickly, and rely on repeatable workflows to ship and iterate ads at high frequency.
Core job-to-be-done: turn product inputs (assets, brief, URL) into high-quality, on-brand ad creatives with lower production overhead.

Interfaces are localized (at minimum **English and Chinese** via `next-intl`); copy and hierarchy should work in both without layout breakage.

### Brand Personality

Brand personality: 简洁, 友好, 智能.
The interface should feel clear and low-friction first, then supportive and approachable, while signaling capable AI assistance without appearing overly technical or intimidating.
Emotionally, the product should create confidence, speed, and control during execution-heavy marketing workflows.

### Aesthetic Direction

Visual direction: clean, modern, practical SaaS with balanced polish.
Reference products: synthesia.io, arcads.ai (for clear information hierarchy, conversion-focused structure, and approachable AI-product presentation).
Anti-references: vercel.com, usemintly.com, remotion.dev, elai.io, quickads.ai.
Design should avoid over-abstract developer aesthetics, over-stylized visual experimentation, or noisy/over-marketed layouts that distract from production tasks.
Theme quality must remain equivalent in light and dark mode (no secondary-quality dark theme).

### Design Principles

1. Clarity over decoration: prioritize fast comprehension, obvious actions, and low cognitive load in every key workflow.
2. Friendly productivity: interactions should feel supportive and human, while preserving efficiency for repeat daily use.
3. Intelligent, not overwhelming: surface AI power through guided defaults, smart recommendations, and clean controls rather than complexity.
4. Campaign-output orientation: optimize screens for shipping ad-ready assets quickly (generate, review, refine, export, iterate).
5. Accessibility as baseline: target WCAG 2.1 AA, ensure strong contrast and keyboard usability, and honor reduced-motion preferences by default.
6. Theme parity: maintain equal visual quality, readability, and hierarchy in both light and dark themes.
7. System fidelity: prefer existing **shadcn/ui + Radix** patterns and **semantic tokens** from `src/styles/global.css` (`primary`, `muted`, `border`, `chart-*`, etc.) over one-off styling; keep motion purposeful and consistent with project conventions.

### Implementation reference (for agents)

- **App**: Next.js App Router, React 19, Tailwind CSS v4 (`src/styles/global.css`, `src/app/styles.css`).
- **UI kit**: shadcn-style components under `src/components/ui`, Radix primitives, `class-variance-authority`, `tailwind-merge` / `cn()`.
- **Data & auth**: Neon Postgres + Drizzle (`@app/db`), Better Auth, product name **App Template** (`package.json`).
- **i18n**: `messages/en.json`, `messages/zh.json`; types via `global.d.ts` / `IntlMessages`.

Do not infer **audience or brand intent** from code alone; when in doubt, re-read this file or ask the product owner.
